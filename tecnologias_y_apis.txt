Basado en la revisión del código y la configuración del proyecto, aquí tienes el desglose completo de las tecnologías, lenguajes y APIs utilizados, tanto para el Frontend como para el Backend:

🖥️ Frontend (Interfaz de Usuario)

Lenguajes y Tecnologías Principales:
* JavaScript (JSX): Es el lenguaje base del frontend.
* React (v19.2): La librería principal para construir las interfaces de usuario.
* Vite (v7.2): Utilizado como herramienta de construcción y servidor de desarrollo para mayor rapidez.
* Tailwind CSS (v3.4): Framework de CSS empleado para dar todos los estilos y el diseño responsivo moderno del sitio.

Librerías Adicionales:
* Framer Motion: Utilizado para manejar las animaciones dinámicas de la interfaz.
* React Router DOM: Para la navegación y manejo de rutas entre las distintas páginas del sitio sin recargar el navegador.
* Recharts: Para la renderización de las gráficas de rendimiento dentro del sistema.
* Axios: Cliente HTTP para hacer las peticiones al Backend y a servicios externos.
* Lucide React: Para la iconografía del sistema.

APIs Externas Utilizadas:
1. CoinGecko API: (api.coingecko.com) Consumida a través de Axios en src/Services/api.js para extraer el mercado actual de criptomonedas y sus variaciones.
2. TradingView Widget: Embebido en scripts dentro de tus componentes (como CryptoChart.jsx) para mostrar gráficas avanzadas interactivas.
3. CryptoLogos API: (cryptologos.cc) Usada como un servicio de imágenes para traer automáticamente los logos de cada criptomoneda (ej. BTC, ETH, USDT) directamente desde sus URLs.

---

⚙️ Backend (Servidor y Base de Datos)

Lenguajes y Tecnologías Principales:
* Python (Mínimo v3.10): El lenguaje de programación empleado para toda la lógica de negocio y conectividad.
* Django (v6.0): El framework web principal que estructura toda la aplicación.
* Django REST Framework (v3.16): Usado en conjunto con Django para construir y servir los endpoints de la API (las conexiones para que tu frontend lea y guarde datos).
* SQLite3: Base de datos relacional ligera, donde se guarda toda la información localmente (db.sqlite3).

Librerías y Módulos Adicionales (Destacados):
* Simple JWT & PyJWT: Implementado para manejar la autenticación y las sesiones de usuario mediante JSON Web Tokens (Tokens de seguridad).
* APScheduler: Utilizado para programar tareas en segundo plano (posiblemente para actualizar periódicamente los precios desde las APIs del mercado de criptos de modo que la DB no se atrase).
* OpenPyXL: Incorporado para generar y leer archivos Excel (muy probable que lo uses para las plantillas o reportes del panel de administrador).
* Yagmail: Para el envío de correos electrónicos automáticos (como el reseteo de contraseñas de los usuarios).
* Pillow: Para permitir el procesamiento y la subida de imágenes (usualmente avatars para los usuarios o bauchers de recarga).

APIs Externas Utilizadas:
1. CoinMarketCap API: Usada en el módulo catalogo/services.py (pro-api.coinmarketcap.com/v1/cryptocurrency/quotes/latest). A diferencia del frontend que usa CoinGecko, tu backend usa CoinMarketCap para cotizar las tasas y precios de compra más exactos y actualizados al momento de realizar transacciones en el núcleo local.
